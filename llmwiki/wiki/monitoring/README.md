# Monitoring

This folder is intended for documents that describe **integrated awareness** — how information from multiple domains (systems + crew + resources) should be combined to detect problems early and support proactive decision making.

Examples of future content:
- Cross-system risk indicators
- Early warning logic
- How to combine crew state with system performance
- Recommended monitoring dashboards or briefing formats for BAU

This category supports AEON's goal of being a proactive cognitive partner rather than a purely reactive system.
